# SovArchitect — Sovereign Azure Architect

> A GitHub Copilot CLI extension that guides you through sovereignty requirements intake and generates complete sovereign Azure architectures with Bicep IaC.

## What it does

SovArchitect is a **Copilot CLI extension** that acts as an expert sovereign workload designer for Microsoft Azure. It:

1. **Asks structured intake questions** across three sovereignty pillars — Data, Control Plane, and Software
2. **Validates** your existing Sovereign Landing Zone posture
3. **Designs** a complete end-to-end sovereign architecture tailored to your profile
4. **Generates Bicep templates** ready to deploy, with sovereignty controls built in
5. **Rubber-Duck** for output validation using a different LLM 
6. **Saves outputs** to a timestamped session folder — every run is isolated

Each session produces:
```
sovereign-output/
  2026-05-01T14-00-00-a3d115/
    architecture.html     ← visual architecture reference
    <workload>.bicep      ← deployable Bicep template
```

## Supported sovereignty profiles

| Dimension | Options |
|---|---|
| **Data residency** | Any Azure region or jurisdiction |
| **Regulations** | GDPR, NIS2, FedRAMP High, ITAR, CJIS, EUCS, ISO 27001, HIPAA |
| **Cloud environment** | Commercial Azure, Azure Government, Sovereign/Dedicated Region, Air-gapped |
| **Key management** | Microsoft-managed, CMK (Key Vault), Managed HSM, On-premises HSM |
| **Portability** | OSS stack, OCI containers, multi-cloud portable designs |

## Prerequisites

- [GitHub Copilot CLI](https://githubnext.com/projects/copilot-cli) installed
- Node.js ≥ 20
- **Experimental mode enabled** — extensions require experimental mode. Enable it once with:
  ```bash
  copilot --experimental
  ```
  or run `/experimental` from within a Copilot CLI session. The setting persists across sessions.

## Installation

The extension lives in a subdirectory of this repo, so a plain `git clone` would place it at the wrong path. Use sparse checkout instead:

```bash
# macOS / Linux — sparse checkout (recommended)
git clone --filter=blob:none --sparse https://github.com/verboompj/SovArchitect /tmp/SovArchitect && \
  cd /tmp/SovArchitect && \
  git sparse-checkout set .copilot/extensions/sovereign-azure && \
  cp -r .copilot/extensions/sovereign-azure ~/.copilot/extensions/ && \
  cd ~/.copilot/extensions/sovereign-azure

# Or use curl to download files directly (no git required)
mkdir -p ~/.copilot/extensions/sovereign-azure && \
  BASE="https://raw.githubusercontent.com/verboompj/SovArchitect/main/.copilot/extensions/sovereign-azure" && \
  curl -fsSL "$BASE/extension.mjs" -o ~/.copilot/extensions/sovereign-azure/extension.mjs && \
  curl -fsSL "$BASE/package.json"   -o ~/.copilot/extensions/sovereign-azure/package.json && \
  cd ~/.copilot/extensions/sovereign-azure
```

```powershell
# Windows (PowerShell) — sparse checkout
git clone --filter=blob:none --sparse https://github.com/verboompj/SovArchitect $env:TEMP\SovArchitect
cd $env:TEMP\SovArchitect
git sparse-checkout set .copilot/extensions/sovereign-azure
Copy-Item -Recurse .copilot\extensions\sovereign-azure "$env:USERPROFILE\.copilot\extensions\sovereign-azure"
cd "$env:USERPROFILE\.copilot\extensions\sovereign-azure"
```

Restart (or reload) the Copilot CLI — the extension loads automatically.

## Usage

Just ask naturally:

```
create a webapp and relational database backend, it has to comply to sovereignty

-or-

create a 3 tier VM setup hosting a web application, it has to comply to sovereignty
```

The extension guides you through:
1. Existing Landing Zone check
2. Data sovereignty intake (residency, regulations, data types, cross-border)
3. Control plane intake (environment, key management, Lockbox)
4. Software sovereignty intake (portability, OSS, vendor independence)
5. Architecture design + Bicep generation

## Architecture

The extension is a single ESM file (`extension.mjs`) with no external runtime dependencies beyond the Copilot SDK. It uses:

- **`@github/copilot-sdk/extension`** — `joinSession`, tools, hooks
- **Node.js `fs` / `path`** — output file writing
- **`fetch`** (built-in Node ≥ 18) — Microsoft Learn documentation search

All architecture content and Bicep is **generated by the LLM at runtime** — the extension provides the state machine, intake forms, sovereignty context, and file persistence.
